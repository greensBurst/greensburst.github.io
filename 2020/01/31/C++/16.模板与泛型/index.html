16.模板与泛型

2020-01-31- 2020-11-22
-
C++-itc

函数模板

  • 使用模板的两种方式
    • 自动类型推导：必须能推出一致的类型
    • 显示指定类型
  • 模板必须确定出T的类型才可以使用
  • 普通函数和函数模板区别
    • 普通函数可以发生隐式类型转换
    • 函数模板
      • 自动类型推导不会发生隐式类型转换
      • 显示指定类型可以发生隐式类型转换
  • 调用规则
    • 如果普通函数和函数模板都可以匹配，优先使用普通函数
    • 如果想强制调用函数模板，需要加空模板参数列表 fun<>(a);
    • 函数模板也可以发生函数重载
    • 如果函数模板可以产生更好的匹配，那么优先使用函数模板
  • 函数模板机制
    • 函数模板并不是万能，对于自定义类型数据，有些情况不能够正常使用
    • 通过函数模板产生的函数，我们称为模板函数
    • 两次编译：对函数模板进行第一次编译，检查语法错误，对调用时替换T之后生成的模板函数，做第二次编译

函数模板与函数重载

类模板

  • 类模板使用时，不可以用自动类型推导，只能显示指定类型
  • 模板中也可以使用默认参数，这样调用时就不用显示指定了
    • template<class A, class B = int>
    • T t(“Tom”,18); //使用
  • 类模板中成员函数并不是一开始创建，而是在替换T后才会生成
  • 类模板做函数参数的三种方式
    • 传入指定类型 fun(A& a);
    • 将参数模板化 template fun(A& a);
    • 将类模板化 template fun(T& a);
  • 子类在继承时，必须确定出父类的T的数据类型，否则无法给T分配内存空间
  • 类模板的成员函数的分文件编写(头文件写类的定义，源文件写函数实现)
    • 类模板成员函数并不是一开始创建，导致直接包含头文件，链接不到
    • 解决方法
      • 直接包含源文件(不推荐)
      • 将类模板中的声明和实现都写到一个文件中
        ，后缀名改为.hpp
