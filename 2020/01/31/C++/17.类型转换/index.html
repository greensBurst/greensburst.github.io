C++-itc

类型转换的问题

  • 子类是大于等于父类的
  • 父转子(向下类型转换)会有风险，子类指针的寻址范围大于等于父类的有效寻址范围，有可能造成越界
  • 子转父(向上类型转换)属于安全转换
  • 如果发生了多态，那么转换总是安全的

C++风格类型转换

  1. 静态类型转换
    • 允许内置数据类型转换
    • 允许父子之间的指针或引用转换(两个非父子类不行)，但向下转换依旧是不安全的
  2. 动态类型转换
    • 是一种严格的转换，不允许有风险
    • 所以不能做内置类型转换(比如double转int会损失精度)
    • 可以子转父
    • 父转子需要发生多态才可以
  3. 常量转换
    • 可以卸掉或加上const修饰
    • 只能用作指针或引用的转换
    • const int a = 1;是在符号表上，int a = 1;是在栈上，所以不行
  4. 重解释转换
    • 最不安全的一种
    • 什么都能转，建议不要用
