3.内联函数

2020-01-31- 2020-11-22
-
C++-itc

宏函数的缺陷：

  1. 需要加小括号保证运算完整性
    • #define ADD(x,y) x+y
    • ret = ADD(1,2) * 3
  2. 即使加了括号有些情况依然与预期不符
    • #define CMP(x,y) (((a)<(b))?(a):(b))
    • a = 10, b = 20
    • ret = CMP(++a,b) // 12

内联函数使用的注意事项

  1. 在函数定义前添加inline关键字，仅在函数原型前使用此关键字无效，定义和实现都要写
  2. 编译器必须能看见内联函数的代码才能在编译期展开，因而内联函数必须实现在头文件中
  3. 在类定义中给出函数体的成员函数自动成为内联函数
  4. 函数体代码量大，或包含循环，不要使用内联
  5. 构造函数和析构函数有可能隐含附加操作，慎用内联
  6. 对于编译器来说，内联函数仅仅是建议，编译器会自主选择是否使用内联函数
