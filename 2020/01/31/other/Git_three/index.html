先让我们看一下现在我们本地工作区的情况。

上一节我们将b2分支的内容合并到了master分支，并解决了冲突，接着我们删掉了b1 b2两个分支。

现在请大家考虑一下情况，假如我们的项目需要多个人开发，我们该如何做到项目组的每个人都能实时共享项目的最新状态呢？又或者只有自己一个人做开发，但如果我们自己的电脑炸了，又没有备份该怎么解决呢？这里我们就要提到远程仓库的概念。

我们可以使用git工具在另一台电脑上创建一个远程的仓库，但很显然这需要我们在准备一台电脑用作仓库使用，但这样做不仅需要成本，而且我们自己的电脑也不稳定，显然这并不是好的选择。我们可以用作项目托管的公司提供的免费的远程仓库，像是github,gitlab,码云等等都可以为我们提供免费的远程仓库。

下面我以github为例，教大家如何创建一个自己的仓库。

1.注册github账号

就正常的填一下用户名，邮箱，密码就好，然后他会给你发个验证邮件，验证之后就注册好了。这里的用户名尽量都用小写的吧，也不是大写不行，就是个人的建议。

2.创建新的仓库


按照上面的流程一步步操作，我们就创建好了自己的远程仓库。上图中https和ssh的区别是前者操作你的仓库的时候需要认证你的github账号密码，后者需要在本地配置ssh再操作仓库就不需要github账号密码了。这里我们先使用https，ssh的配置会在之后发布。

git remote add name https://命令可以添加一个远程仓库，需要你指定仓库名字和地址。git remote可以查看你添加的远程仓库。git push -u name branch命令第一次同步本地到远端需要指定name：名字和branch：分支。在之后的同步操作只需要git push就可以了。

现在让我们去github上查看我们的项目是否已经同步到了远端。

可以看到我们的123.txt文件已经成功的在github上显示出来了。

假如我们的伙伴小红觉得我们的代码写的非常好，想拿到我们的代码学习一下，那么小红该如何获取我们远程仓库的代码呢？

经过小红的一番操作，她终于拿到了项目的代码。git pull命令可以拉取指定仓库指定分支的内容。但小粉觉得小红的操作太复杂了，让我们看看小粉是如何操作的。

小粉直接使用了git clone https://命令克隆了test仓库到本地，可以在test目录下看到我们的123.txt。

我们也可以随时删除我们在github上的仓库。

github可能还会提示你输入账户和密码，输入后就可以成功删除你的仓库了。

恭喜你，现在你已经学会git和github的基础使用了！
