流与文件

C++-thuC++

1. 标准流类库

流：数据从源到目的的流动

1.1 输入输出流类

  1. 输入输出流：iostream
  2. 输入流：istream
  3. 输出流：ostream

1.2 字符串流类

  1. 输入输出字符串流：strstream
  2. 输入字符串流：istrstream
  3. 输出字符串流：ostrstream

1.3 文件流类

  1. 输出文件流：ofstream
  2. 输入文件流：ifstream
  3. 输入输出文件流：fstream

1.4 全局流对象

  1. std::cout：标准输出流对象，一般对应标准输出设备
  2. std::cin：标准输入流对象，一般对应标准输入设备
  3. std::cerr：标准错误流对象，一般对应标准错误输出设备
  4. std::clog：标准日志流对象，一般对应标准日志输出设备

std::cout、std::cerr与std::clog为std::ostream类的对象；std::cin为std::istream类的对象

1.5 插入与提取

插入

  • 目的：将数据对象插入到流中
  • 插入操作符也称输出操作符
  • std::cout << “Hello World!”; 将字符串插入到输出流

提取

  • 从流中提取数据对象
  • 提取操作符也称输入操作符
  • int a; std::cin >> a; 从输入流中提取整数

注意：因为流可能被重定向或束定，有时使用输入输出描述流操作可能会让人迷惑

1.6 常用输入输出流函数

  1. 判断流是否已结束：cin.eof()
  2. 读取单个字符：cin.get( istream::char_type & c )
  3. 读取字符串：cin.get( istream::char_type * s, streamsize n, istream::char_type delimiter = '\n' )
  4. 读取单行文本：cin.getline( istream::char_type * s, streamsize n, istream::char_type delimiter = '\n' )
  5. 输出单个字符：cout.put( ostream::char_type c )

2. 流格式

2.1 流格式标志

2.2 位掩码

  • 使用32位整数的位代表流格式标志：ios_base::fmtflags
  • 每个标志位可单独设置与清除

2.3 设置预定义标志位

  • cout.setf( ios_base::showbase ); 输出整数前缀，十六进制前添加“0x”

2.4 清除预定义标志位

  • cout.unsetf( ios_base::showbase ); 清除上述标志

2.5 位组：特定标志位集合，位组中的标志位互相排斥

  • ios_base::adjustfield、ios_base::basefield与ios_base::floatfield
  • cout.setf( ios_base::hex, ios_base::basefield ); 设置十六进制输出格式，使用单参数版本无效果

2.6 设置用户自定义参数：单参数版本

  • 设置时传递用户指定值
  • std::cout.width( 8 ); 将最小输出宽度定为8个字符
  • std::cout.precision( 8 ); 将输出精度定为8位
  • std::cout.fill( '?' ); 使用‘?’填充空白字符位置

2.7 获取用户自定义参数：无参数版本

  • 获得当前的输出精度值：std::streamsize precision = std::cout.precision();

3. 操纵符

3.1 操纵符的目的：控制流的输入输出格式

  • 无参数操纵符：函数指针
  • 单参数操纵符：函子，即带有函数指针功能的操纵符类的对象，实现上为重载了函数调用操作符的操纵符类

3.2 操纵符示例

  • 头文件：“iomanip”
  • std::cout << "Hello World!" << std::endl;
  • int n = 1024; std::cout << std::dec << n << '\n' << std::hex << n << std::endl;

4. 文件流

4.1 文件特性

  • 文件一般保存在外部存储设备上
  • 文件生命周期可能远远超过创建它的程序本身

4.2 文件操作：读、写

  • 一般使用文件指针，该指针代表文件的当前访问位置
  • 老式的C语言使用文件句柄（handle）或文件描述符（file descriptor）表示某个打开的文件数据对象

4.3 文件流的使用

  • 头文件：“fstream”
  • 按照特定格式重载类的流操作符
  • 创建文件流对象，输入输出

4.4 文件打开模式

  • std::ios_base::app：每次插入都定位到文件流的尾部
  • std::ios_base::binary：使用二进制而不是文本格式打开文件流
  • std::ios_base::in：流用于输入目的，允许提取，此为std::ifstream流缺省设置
  • std::ios_base::out：流用于输出目的，允许插入，此为std::ofstream流缺省设置
  • std::ios_base::trunc：若文件存在，清除文件内容，此为std::ofstream流缺省设置
  • std::ios_base::ate：若文件存在，定位到文件尾部文件一般保存在外部存储设备上

5. 流状态

5.1 流状态：表示操作成功或失败的状态信息

  • std::ios_base::goodbit：流完好无损
  • std::ios_base::badbit：流已出现致命错误，一般无法恢复
  • std::ios_base::eofbit：流结束时设置
  • std::ios_base::failbit：流操作失败时设置，可能恢复

5.2 流状态对流操作行为的影响

  • 一旦流状态存在错误，所有I/O操作都失效
  • 在出现std::ios_base::failbit与std::ios_base::badbit状态时，输出操作立即停止
  • 在非std::ios_base::goodbit状态时，输入操作立即停止

5.3 流状态测试

  • bool std::ios_base::good() const：没有出现任何错误时返回真
  • bool std::ios_base::eof() const：设置std::ios_base::eofbit状态时返回真
  • bool std::ios_base::fail() const：设置std::ios_base::failbit状态时返回真
  • bool std::ios_base::bad() const：设置std::ios_base::badbit状态时返回真
  • bool std::ios_base::operator !() const：与std::ios_base::fail() 效果相同
  • std::ios_base::operator void*() const：std::ios_base::fail() 为真时返回空指针，否则非空

6. 流定位

6.1 流位置指针

  • 位置指针指向下一次读写操作时的数据对象在流中的位置，该指针会随着输入输出操作而不断变化
  • 单向流：一个位置指针；双向流：两个位置指针

6.2 流位置指针的获取

  • 成员函数tellp()：获取当前的流位置指针（写指针）
  • 成员函数tellg()：获取当前的流位置指针（读指针）

6.3 流位置指针的定位

  • 成员函数seekp()：将文件位置指针定位到某个特定位置，用于插入（输出）目的
  • 成员函数seekg()：将文件位置指针定位到某个特定位置，用于提取（输入）目的的定位

6.4 流定位函数seekp()与seekg()

  • 单参数版本：可以使用获取的位置指针
  • 双参数版本：第一个参数为偏移量；第二个参数为定位基准

6.5 定位基准

  • std::ios_base::beg：从流的开始位置开始计算偏移量
  • std::ios_base::cur：从当前位置开始计算偏移量
  • std::ios_base::end：从流的结束位置开始计算偏移量

补充

  1. 标准输入流

  • cin.get(); 一次只能读取一个字符
  • cin.get(buf,sizeof(buf)); 读取字符串，换行符会遗留在缓冲区中
  • cin.getline(buf,sizeof(buf)); 读取字符串，换行符不会遗留在缓冲区中，也不会读入
  • cin.ignore(); 默认忽略1个缓冲区的字符，可以填写忽略个数
  • cin.peek(); 查看缓冲区第一个字符，不取走
  • cin.putback(c); 将字符放回缓冲区，并且放回原位
  • cin.fail(); 查看缓冲区标志位，0为正常，1为异常，比如int n; cin >> n; 但是缓冲区之只能读到字符，标志位就会异常
  • cin.clear(); 重置标志位
  • cin.syna(); 清空缓冲区

  1. 标准输出流

  • cout.put(); 向缓冲区写字符
  • cout.write(buf,sizeof(buf)); 从buf中写n个字节到当前输出流
  • 通过流成员函数格式化输出
  • 通过控制符格式化输出
