sql语句在mysql中是如何查找的？

hash查找？暴力？索引？

索引的数据结构有哪些？数组、链表、红黑树、哈希表，b树(b树，b+树)

数据量大的时候数组和链表是不行的

范围查找的时候，hash是不行的，部分索引查询也不行

什么时候hash？查询条件不会变，而且没有部分查询和范围查询

为什么红黑树不可以？

数据量大的时候，红黑树的io操作也很大

而且磁盘按页读取，但每次只能用到一个数据单位，一页16k，但一个数据可能只有4字节，很浪费

但是C++的map可以用红黑树，因为它是存在内存的

磁盘io次数过多？ -> N叉树

读取浪费？ -> 连续空间，每个节点是一个线性表

b树的性质：

  1. m阶b树最多含m个指针，m-1个关键字，m>=2
  2. 除根节点和叶子节点，其他每个节点至少有m/2个指针
  3. 若根节点不是叶子节点，则至少有两个指针

b树的操作：

  1. 分裂

b树适合作mysql的索引么？

b树不适合范围查找，还是会有树的遍历操作，而且每一个key都对应一条记录的地址，存储开销大

这些问题可以用b+树解决

select * from student where id = 0 and name = "张三"

建立id的索引，先找到id=0，然后找name=张三，这是索引建立的最左原则，当然也可以建立id+name的索引

不过索引不是建的越多越好，如果数据有改动，维护索引很麻烦的

即便查询不涉及到索引字段，也是会走索引的叶子节点进行遍历查找的，比全表搜索还是要快一些
