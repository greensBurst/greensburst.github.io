gdb

调试准备

  • 通常在调试编译时，会关掉编译器优化”-O0”，打开调试选项”-g”，打开警告提示”-Wall”，避免不必要的bug
  • “-g”的作用是在可执行文件中加入源代码的信息，比如可执行文件中第几条机器指令对应源代码的第几行，但不是把整个源文件嵌入到可执行文件中，所以调试时必须保证gdb能找到源文件

调试命令

启动和退出

  • 启动：gdb 可执行程序
  • 传递参数：(gdb) set args 8 9
  • 显示参数：(gdb) show args
  • 退出：(gdb) quit

查看代码

  • 当前文件(l === list)
    • (gdb) l
    • (gdb) l 行号
    • (gdb) l 函数名
  • 非当前文件(l === list)
    • (gdb) l 文件名:行号
    • (gdb) l 文件名:函数名
  • 查看默认显示行数(list === listsize)
    • (gdb) show list
  • 更改显示行数(list === listsize)
    • (gdb) set list 行数

断点操作(b === break)

设置断点

  • (gdb) b 行号
  • (gdb) b 函数名
  • (gdb) b 文件名:行号
  • (gdb) b 文件名:函数名

查看断点(i === info)

  • (gdb) i b

删除断点(d === del === delete)

  • (gdb) d 断点编号

设置断点无效(dis === disable)

  • (gdb) dis 断点编号

无效断点生效(ena === enable)

  • (gdb) ena 断点编号

设置条件断点

  • (gdb) b 行号 if 条件语句

调试命令

运行

  • (gdb) start:只运行一行
  • (gdb) run:运行直到断点

运行到下一个断点(c === continue)

  • (gdb) c

打印变量(p === print)

  • (gdb) print 变量名
  • (gdb) print/x 变量名 十六进制
  • (gdb) print/o 变量名 八进制

打印变量类型

  • (gdb) ptype 变量名

单步调试

  • 会进入函数体(s === step)
    • (gdb) s
    • 跳出函数体：(gdb) finsh
      • 函数体内不能有断点
  • 不会进入函数体(n === next)
    • (gdb) n
  • 变量自动显示
    • (gdb) display 变量名
    • 查看自动打印哪些变量(i === info)：(gdb) i display
  • 取消变量自动显示
    • (gdb) undisplay 编号
  • 设定变量值
    • (gdb) set var 变量名=值
  • 跳出循环
    • (gdb) until
    • 循环体内不能有断点
    • 执行完循环体最后一行才会跳出

多进程调试

  • 默认跟踪父进程
  • 设置跟踪父进程：set follow-fork-mode parent
  • 设置跟踪子进程：set follow-fork-mode child

core文件

  • $ ulimit -a查看core文件大小，为0是不会创建core文件的
  • $ ulimit -c 1024改变core文件大小
  • gcc main.c -g生成可调试程序
  • 运行程序如果出错了就会生成core文件
  • (gdb) core-file core查看错误信息
