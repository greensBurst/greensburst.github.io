进程控制

每个进程结束之后，都会释放自己虚地址空间中用户区的数据，内核区的pcb没办法自己释放，需要父进程释放。

结束进程

孤儿进程

  • 父进程先结束，子进程就成为孤儿进程
  • PID为1的进程会成为该孤儿的父进程，负责回收子进程的pcb

僵尸进程

  • 子进程先结束，但父进程没有做回收工作，子进程成为僵尸进程
  • 僵尸进程没有用户区，只有一个pcb，不能称为一个h存活的进程
  • 僵尸进程不能被kill -9杀死

进程回收

  • pid_t wait(int *status);

    • 是一个阻塞函数
    • 当前进程还有子进程时，调用wait会阻塞
    • 子进程被全部回收，调用wait不会阻塞
    • wait调用一次只能回收一个子进程
    • status：记录子进程退出时的状态
      • 正常退出：走的exit函数，return下面也是exit
      • 异常退出：被信号干掉
    • 返回值：
      • 大于零：被回收的子进程PID
      • -1：调用失败

  • pid_t waitpid(pid_t pid, int *status, int options);

    • pid:
      • >0:某个子进程PID
      • =0:回收当前进程组的子进程
      • -1:回收所有子进程 == wait(NULL)，用的最多
      • <0:某个进程组ID，回收指定进程组子进程
    • option:
      • 0:阻塞
      • WNOHANG:非阻塞
    • 返回值:
      • >0:回收的子进程PID
      • =0:options = WNOHANG，还有子进程存活
      • -1:错误，已经没有子进程了
