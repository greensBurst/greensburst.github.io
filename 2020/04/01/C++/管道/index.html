管道

管道的本质？

  • 内核缓冲区
  • 拥有文件特质，可以读写
  • 匿名管道：没有文件的实体
  • 具名管道：有文件实体，不存数据
  • 可以使用文件操作处理管道

匿名管道

匿名管道的特点？

  • 在磁盘上没有实体，是内存中的一块缓冲区
  • 这个缓冲区是fork前得到的
  • 内核缓冲区有两部分
    • 读端 -> 可读文件描述符
    • 写端 -> 可写文件描述符
  • 父进程被销毁，管道被释放
  • 默认阻塞

管道的原理？

  • 循环队列
  • 默认容量4k，可以使用ulimit -a查看

匿名管道的局限性

  • 管道中的数据只能读一次
  • 半双工：数据是单向流动的
  • 只能用于有亲缘关系的进程通信

创建匿名管道

  • 参数:
    • pipefd: 传出参数
    • pipefd[0] -> 管道的读端
    • pipefd[1] -> 管道的写端
  • 返回值:
    • 0: 调用成功
    • -1: 失败
  • 一般读端会关闭fd[2]，写端会关闭fd[1]

案例

  • 父子进程间通信，实现 ps aux
  • 子进程 -> ps aux
  • 子进程得到的结果，给到父进程
  • 涉及到有血缘关系的进程通信 -> pipe
  • 创建进程 -> fork()
  • 子进程做的事儿: execlp()，执行命令数据会默认写到终端
    • 重定向: stdout_fileno->管道的写端 dup2

管道的读写行为

  • 读操作

    • 如果管道中有数据
      • read读，read返回值是读到的字节数
    • 管道中没有数据
      • 写端没有关闭，但是写的慢
        • read阻塞
      • 写端关闭了
        • read解除阻塞，返回值为0
          • 数据读完了

  • 写数据

    • 管道有空间
      • 接着写，写满之后阻塞
    • 没有空间
      • 直接阻塞
    • 写数据的时候，读端关闭了
      • 管道破裂
      • 进程被 SIGPIPE 信号杀死

  • 设置管道非阻塞(默认两端都是阻塞的)

具名管道

特点

  • 在磁盘上有一个文件 -> 伪文件
    • 通过这个伪文件给不同 的进程搭建一个桥梁 -> 找到同一块内核缓冲区
    • 这个磁盘文件大小，永远为0
  • 内核缓冲区 -> 环形队列实现的
  • 数据只能被读一次
  • 两端默认阻塞

使用场景

  • 有血缘关系的进程间通信
  • 没有血缘关系的进程通信

创建方式

  • shell命令：$ mkfifo 名字
  • 系统函数：
    • pathname: 创建的管道文件对应路径和名字
    • mode: 用户对管道文件的操作权限，八进制的数，最终权限: (mode & ~umask)
