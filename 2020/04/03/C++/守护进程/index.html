守护进程

Daemon Process，又叫精灵进程，是linux中的后台服务进程，生存期较长，通常独立于控制终端并周期性的执行某种任务，或等待处理某些发生的事件，一般用d作为名字的结尾，比如mysqld，httpd…

守护进程的特点

  • 名字一般以d结尾
  • 脱离终端，没有输入输出
  • 周期性执行某些特定操作

进程组

一个及以上进程的集合，在linux中所有进程都属于一个进程组，默认父子进程在同一个进程组中

  • 进程组的第一个进程就是组长
  • 进程组ID和组长进程ID相同

    getgrp

  • pid_t getgrp(void);
  • posix标准，获取当前进程所在进程组ID

getpgid

  • pid_t getpgid(pid_t pid);
  • bsd标准，获取指定进程所在进程组ID

setpgid

  • int setpgid(pid_t pid, pid_t pgid);
  • 可以将一个进程送到另一个进程组
  • 也可以创建一个新进程组，pgid = pid

会话

一个及以上进程组的集合

  • 创建会话的注意事项
    • 该进程不能为进程组长
    • 创建会话的进程成为新进程组组长
    • 某些linux发行版需要root权限
    • 创建出的新会话会丢弃原有控制终端
    • 一般步骤：fork -> 父进程死亡 -> 子进程创建会话

getsid

  • pid_t getsid(pid_t pid);
  • include <unistd.h>
  • 获取进程所在会话ID
  • 成功返回会话ID，失败返回-1

setsid

  • pid_t setsid(void);
  • include <unistd.h>
  • 当前进程会被提升为会话，当前进程不能为进程组长
  • 当前进程会脱离原控制终端

创建守护进程

  1. 父进程创建子进程，杀死父进程（必需）
  2. 将子进程提升为会话（必需）
  3. 修改进程工作目录，切换到不可卸载的目录中（非必需）
    • chdir();
    • 如果工作目录在u盘，是可以被卸载的
  4. 修改umask掩码（非必需）
    • umask();
  5. 关闭或重定向文件描述符（非必需）
    • 关闭：close(0); close(1); close(2);
    • 重定向：int fd = open(“/dev/null”,O_RDWR); dup2(0,fd); dup2(1,fd); dup2(2,fd);
  6. 写核心操作流程（必需）
