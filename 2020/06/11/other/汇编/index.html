    1 汇编语言基础

    1.1 寄存器

    32位通用寄存器*8

    • EAX 累加器，用于算术、逻辑运算
    • EBX 基址寄存器，用于存放存储器地址，以方便指向变量或数组中的元素
    • ECX 计数器，用于循环操作等指令中的计数器
    • EDX 数据寄存器，用于存放数据
    • ESI 源变址寄存器，用于指向字符串或数组的源操作数
    • EDI 目的变址寄存器，用于指向字符串或数组中的目的操作数
    • EBP 基址指针寄存器，默认指向堆栈区域数据，主要用于在子程序中访问通过堆栈传递的参数和局部变量
    • ESP 堆栈指针寄存器，专用于指向堆栈区域顶部数据，在涉及堆栈操作的指令中会自动增加或减少

      16位段寄存器*6

    • CS 代码段寄存器
    • SS 堆栈段寄存器
    • DS 数据段寄存器
    • ES 附加段寄存器，也是用于存放数据的数据段
    • FS 数据段性质寄存器
    • GS 数据段性质寄存器

      32位标志寄存器*1（8086）

      除了标志位目的操作数的标志传送指令外，其他传送指令并不影响标志
    • 状态标志
      • CF 进位标志：当无符号整数加减运算结果的最高有效位有进位或借位时，将CF置1
      • PF 奇偶标志：反应运算结果最低字节中1的个数是奇是偶，偶数则置PF为1
      • AF 调整标志：反应加减运算时最低半字节有无进位或借位，有则置AF为1，该标志位由CPU使用，用户不必关心
      • ZF 零标志：反映运算结果是否为0，为0置ZF为1
      • SF 符号标志：反应运算结果正负，最高位为1即负数则置SF为1
      • OF 溢出标志：当有符号整数加减运算的结果超出范围，将OF置1，有符号数加减运算只有没有溢出时结果才是正确的。当两个相同符号数相加，而运算结果与原数据相反才会溢出，其他情况不会溢出
    • 控制标志
      • DF 方向标志，仅用于串操作指令中，控制地址的变化方向
    • 系统标志
      • TF 陷阱标志，用于控制处理器是否进入单步操作方式
      • IF 中断允许标志，用于控制外部可屏蔽中断是否可以被处理器响应

        32位指令指针寄存器*1

    • EIP 保存存储器地址，即将要执行的指令在主存中的位置，具有自动增量的能力，不能像通用寄存器一样直接修改

    1.2 输入输出子程序库

    • DISPCRLF printf(“\n”)
    • DISPRD 显示8个通用寄存器内容
    • DISPRF 显示6个状态标志状态
    • DISPHD printf(“%x”, eax) 16进制显示
    • DISPUID printf(“%u”, eax) 无符号10进制
    • DISPSID printf(“%d”, eax) 有符号10进制
    • READMSG scanf(“%s”, eax) EAX存地址，调用后地址存放字串，EAX存长度不含0
    • READC scanf(“%c”, al) 调用后AL存字符ASCII码
    • READHD scanf(“%x”, eax) 输入8位十六进制数据，存在EAX中
    • READUID scanf(“%u”, eax) 输入无符号十进制整数，存在EAX中
    • READSID scanf(“%d”, eax) 输入有符号十进制整数，存在EAX中

    2 通用数据处理指令

    2.1 数据传送类指令

    2.1.1 传送指令 MOV

    • mov reg/mem, imm
      • mov eax, 200
      • mov bvar, byte ptr 200
      • mov [ebx], word ptr 200
    • mov reg/mem/seg, reg
      • mov eax, ebx
      • mov bvar, al
      • mov [ebx], cx
      • mov ds, bx
    • mov reg/seg, mem
      • mov al, bvar
      • moc dx, [ebx]
      • mov edx, dvar[edi]
      • mov ds, wvar
    • mov r16/m16, seg
      • mov ax, ds
      • mov wvar, ds
      • mov [ebx], ss

    2.1.2 交换指令 XCHG

    • XCHG reg, reg/mem
    • XCHG reg/mem, reg

    2.1.3 堆栈操作指令

    栈底对应高地址，栈顶对应低地址

    进栈指令 PUSH

    • PUSH r16/m16/i16/seg
    • PUSH r32/m32/i32

    出栈指令 POP

    • POP r16/m16/seg
    • POP r32/m32

    2.1.4 EBP

    既然堆栈是利用主存实现的，我们当然就能以随机存取方式读写其中的数据，通用寄存器之一的堆栈基址指针EBP就是出于这个目的而设计的

    2.1.5 其他传送指令

    地址传送指令 LEA

    存储器操作数具有地址属性，利用地址传送指令可以获取其地址，其中最常用的是获取有效地址指令LEA，该指令在执行时计算偏移地址

    • LEA r16/r32, mem
    • LEA eax, [ebx+ecx] ;利用LEA实现加法操作

    换码指令 XLAT

    使用XLAT指令前，需要将EBX指向数据表首地址，并给AL赋值数据项的偏移量，执行的功能是将数据表该位置的数据取出赋值给AL，可以表达为AL=[EBX+AL]，默认使用DS数据段

    标志传送指令

    • CLC：复位进位标志，CF=0
    • STC：置位进位标志，CF=1
    • CMC：取反进位标志
    • CLD：复位方向标志，DF=0，串操作后地址增大
    • STD：置位方向标志，DF=1，串操作后地址减小
    • CLI：复位中断标志，IF=0，禁止可屏蔽中断
    • STI：置位中断标志，IF=1，允许可屏蔽中断
    • LAHF：标志寄存器低字节内容传送到AH寄存器
    • SAHF：AH寄存器内容传送到标志寄存器低字节
    • PUSHF：标志寄存器低16位压入堆栈
    • POPF：堆栈顶部一个字量数据弹出到标志寄存器低16位
    • PUSHFD：32位标志寄存器全部内容压入堆栈
    • POPFD：当前堆栈顶部一个双字数据弹出到标志寄存器

    2.2 算数运算类指令

    2.2.1 加法指令

    加法指令 ADD

    • ADD reg, imm/reg/mem
    • ADD mem, imm/reg

    带进位加法指令 ADC

    ADC除了完成ADD加法运算外，还要加上进位CF

    • ADC reg, imm/reg/mem
    • ADC mem, imm/reg

    增量指令 INC

    不影响CF，但影响其他状态标志位

    • INC reg/mem

    2.2.2 减法指令

    减法指令 SUB

    • SUB reg, imm/reg/mem
    • SUB mem, imm/reg

    带借位减法指令 SBB

    SBB除了完成SUB减法运算外，还要减去借位CF,64位减法先减低32位，再减高32位

    • SBB reg, imm/reg/mem
    • SBB mem, imm/reg

    减量指令 DEC

    不影响CF，但影响其他状态标志位

    • DEC reg/mem

    求补指令 NEG

    即用零减去操作数，结果返回操作数，与用0做减法的SUB指令一样，可用于对负数求补码或由负数补码求绝对值

    • NEG reg/mem

    比较指令 CMP

    使目的操作数减去源操作数，差值不回送到目的操作数，但按照减法结果影响状态标志

    • CMP reg, imm/reg/mem
    • CMP mem, imm/reg

    2.2.3 乘除法指令

    IA-32乘除法需要区别有无符号，并隐含使用EAX和EDX，有点复杂而且好像不咋用乘除法这里没写，页数76

    零位扩展和符号扩展指令

    位数增加，大小不变

    • 无符号数零位扩展 MOVZX 高位补0
      • MOVZX r16, r8/m8
      • MOVZX r32, r8/m8/r16/m16
    • 有符号数符号扩展 MOVSX
      • MOVSX r16, r8/m8
      • MOVSX r32, r8/m8/r16/m16

    10进制调整指令

    貌似不重要，连个例子都没有，页数79

    2.3 位操作类指令

    2.3.1 逻辑运算指令

    逻辑与 AND

    结果返回到目的操作数，会设置CF=OF=0，影响SF，ZF，PF

    • AND reg, imm/reg/mem
    • AND mem, imm/reg

    逻辑或 OR

    结果返回到目的操作数，会设置CF=OF=0，影响SF，ZF，PF

    • OR reg, imm/reg/mem
    • OR mem, imm/reg

    逻辑非 NOT

    不影响标志位

    • NOT reg/mem

    逻辑异或 XOR

    结果返回到目的操作数，会设置CF=OF=0，影响SF，ZF，PF，可用XOR EAX,EAX做清零运算

    • XOR reg, imm/reg/mem
    • XOR mem, imm/reg

    测试指令

    将两个数进行逻辑与运算，但不返回到目的操作数

    • TEST reg, imm/reg/mem
    • TEST mem, imm/reg

    2.3.2 移位指令

    移位指令

    影响CF，SF，ZF，PF，如果移动一位，按照最高符号位是否改变设置OF，有改变则OF置1，移动次数大于1，OF不确定，逻辑移位可实现乘除2^N，余数由CF反应

    • 逻辑左移i8/CL位 SHL reg/mem, i8/CL 最低位补0，最高位进CF
    • 逻辑右移i8/CL位 SHR reg/mem, i8/CL 最高位补0，最低位进CF
    • 算数左移 SAL reg/mem, i8/CL 与SHL是同一条指令
    • 算数右移i8/CL位 SAR reg/mem, i8/CL 最高位不变，最低为进CF

    循环移位指令

    从一端移出返回到另一端形成循环，影响CF，OF，OF和移位指令一样

    • 不带进位循环左移 ROL reg/mem, i8/CL 最高位进入CF和最低位
    • 不带进位循环右移 ROR reg/mem, i8/CL 最低位进入CF和最高位
    • 带进位循环左移 RCL reg/mem, i8/CL CF进入最低位，最高位进入CF
    • 带进位循环右移 RCR reg/mem, i8/CL CF进入最高位，最低位进入CF

    3 程序结构

    3.1 分支程序结构

    汇编语言需要首先利用比较CMP、测试TEST、加减运算、逻辑运算等影响状态标志的指令形成条件，然后利用条件转移指令判断由标志表达的条件，并根据标志状态控制程序转移到不同的程序段。

    3.1.1 无条件转移指令 JMP

    • JMP label
    • JMP r32/r16
    • JMP m32/m16
    • JMP m48/m32

    3.1.2 条件转移指令 JCC

    为区别有符号数的大小关系，无符号数的大小关系用高低表示，他需要利用CF确定高低，利用ZF确定相等。两个无符号数据的高低分成四种关系，对应9-12条。判断有符号数的大小需要组合OF和SF标志，并利用ZF标志确定相等于否，两个有符号数的大小也分四种关系，对应13-16条。对于判断是否相等，无论有无符号都是用JZ/JE。

    • JZ/JE ZF=1 等于零/相等
    • JNZ/JNE ZF=0 不等于零/不相等
    • JS SF=1 符号为负
    • JNS SF=0 符号为负
    • JP/JPE PF=1 1的个数为偶
    • JNP/JPO PF=0 1的个数为奇
    • JO OF=1 溢出
    • JNO OF=0 无溢出
    • JC/JB/JNAE CF=1 进位/低于/不高于等于
    • JNC/JNB/JAE CF=0 无进位/不低于/高于等于
    • JBE/JNA CF=1或ZF=1 低于等于/不高于
    • JNBE/JA CF=0且ZF=1 不低于等于/高于
    • JL/JNGE SF!=OF 小于/不大于等于
    • JNL/JGE SF=OF 不小于/大于等于
    • JLE/JNG SF!=OF或ZF=1 小于等于/不大于
    • JNLE/JG SF=OF且ZF=0 不小于等于/大于
