db-redis

redis在6.0版本之前使用单线程处理所有请求。

0. redis的应用场景

Redis 是一个开源（BSD许可）的，内存中的数据结构存储系统，它可以用作数据库、缓存和消息中间件。

上段引用自redis官网，redis不只是数据库，也可以作缓存和消息中间件。

1. redis默认有16个数据库

可以在redis.conf中查看databases项为16，默认使用0号数据库。

2. 默认端口号6379的来历

6379是一个女明星MERZ的九宫格数字。

3. 使用单线程处理请求的原因

多线程主要是为了避免cpu等待io操作，redis的数据都在内存中，出于对线程切换的考虑，我们允许内存io带来的cpu资源浪费，因此单线程的效率会更高。

官方解释是，redis基于内存操作，性能瓶颈不是cpu，而是内存和网络带宽，所以就用单线程了。

4. 一些命令

  • select db切换数据库
  • dbsize查看当前数据库大小
  • flushdb清空当前数据库
  • flushall清空所有数据库
  • keys *查看所有key
  • exists key判断key是否存在
  • move key db移动key到其他数据库
  • expire key seconds设置key失效时间
  • ttl key查看key有效时间
  • del key删除key
  • type key查看key的类型
